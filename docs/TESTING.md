# Тест-план Lar’s Cloud

Перед публічним релізом виконайте цей план на чистій Windows 10 x64 і Windows 11 x64 під звичайним (не адміністраторським) користувачем після встановлення.

## 1. Installer

- Запускається один `LarsCloud_Setup.exe`.
- Програма є в Program Files, меню Пуск, на робочому столі (якщо прапорець вибрано).
- У Programs and Features є Lar’s Cloud та коректний uninstall.
- Після перезавантаження програма запускається у треї.
- На ПК без Python і без .NET Desktop Runtime застосунок працює.

## 2. OAuth

- Обидві кнопки першого екрана відкривають системний браузер.
- Скасування згоди показує зрозуміле повідомлення й не закриває програму.
- Успішний вхід показує правильний email.
- Після перезапуску повторний вхід не потрібний.
- `auth.dat` не містить читабельного access/refresh token.
- «Вийти» відкликає сесію; «інший акаунт» дозволяє вибрати інший Google-профіль.
- Відкликаний у Google Account токен приводить до пропозиції повторного входу.

## 3. Папка

- Можна вибрати довільну папку з пробілами та кирилицею.
- Кнопка створення робить `Lar's Cloud` на Desktop і одразу вибирає її.
- Видалення/переміщення папки показує помилку.
- Недоступний файл не завершує аварійно весь UI; помилка є в історії/журналі.
- Junction/symlink не створює нескінченного обходу.

## 4. Перша синхронізація

Підготуйте вкладені папки, порожній файл, фото, PDF і файл понад 20 МБ.

- На Drive створено `Lar's Cloud / НАЗВА_ПК`.
- Структура підпапок збігається.
- Розмір/кількість/прогрес відображаються.
- Великі файли завантажуються частинами, UI реагує.
- Після завершення всі файли відкриваються і мають правильний розмір.

## 5. Інкрементальна синхронізація

- Без змін повторна спроба завантажує 0 файлів.
- Новий файл завантажується один раз.
- Змінений файл оновлюється з тим самим Drive ID, дублікат не з’являється.
- Однакові імена у різних підпапках не конфліктують.
- Змінений лише час файлу повторно звіряється.

## 6. Видалення

- За замовчуванням видалення локального файлу залишає Drive-копію.
- Після явного ввімкнення опції локально видалений відомий файл видаляється з Drive.
- Програма не видаляє сторонні файли поза власним деревом/SQLite-станом.

## 7. Помилки й відновлення

- Вимкніть мережу до старту: червоний стан, без crash.
- Вимкніть мережу посеред великого upload: є повторні спроби/помилка, наступний запуск завершує резервування без дубліката.
- Заповніть Drive: показано «Недостатньо місця».
- Закрийте Windows під час процесу: після запуску стан бази коректний.
- Спробуйте запустити другий екземпляр: показано повідомлення, друга копія завершується.
- Натисніть хрестик: вікно зникає, tray і scheduler лишаються.
- «Вийти з програми» в tray повністю завершує процес.

## 8. Графік

- Для 1–7 днів правильно змінюється Next sync.
- «Синхронізувати зараз» не залежить від графіка.
- Призупинення скасовує поточний процес і не запускає фоновий.
- Відновлення повертає планувальник.
- Невдала фонова спроба не повторюється частіше ніж раз на 15 хвилин.

## 9. Журнал і сповіщення

- Є щонайменше три останні записи.
- Помилка містить зрозумілий текст.
- У логах немає `access_token`, `refresh_token` і Bearer token.
- Після перевищення 2 МБ лог ротується, зберігається не більше п’яти архівів.
- Перемикачі сповіщень працюють.

## 10. Оновлення

- Старіша версія бачить новий Release і release notes.
- Невірний SHA-256 блокує запуск Installer та видаляє `.download`.
- GitHub offline не закриває програму.
- Після тихого update запускається нова версія, акаунт, папка, історія та налаштування збережені.

## Автоматичні тести

```powershell
dotnet test LarsCloud.sln -c Release
```

Автоматичні тести є базовою перевіркою логіки; OAuth, Drive, tray, Installer та Windows Restart Manager обов’язково перевіряються інтеграційно за планом вище.
