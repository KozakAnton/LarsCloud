# Політика конфіденційності Lar’s Cloud

Дата набрання чинності: 14 липня 2026 року.

Lar’s Cloud — локальна Windows-програма для резервного копіювання файлів у Google Drive користувача.

## Які дані використовуються

- email та ім’я Google-акаунта — лише для показу підключеного профілю;
- OAuth access/refresh token — для дозволених операцій Google Drive;
- шлях, розмір, час зміни й SHA-256 локальних файлів — для визначення змін;
- технічна історія синхронізацій і повідомлення про помилки.

## Як передаються файли

Файли передаються безпосередньо з комп’ютера користувача до Google Drive через офіційний Google Drive API. Lar’s Cloud не використовує власний або сторонній проміжний сервер для файлів.

## Зберігання

OAuth-токен зберігається локально та шифрується Windows DPAPI для поточного Windows-користувача. Налаштування, SQLite-стан і технічний журнал зберігаються у `%LOCALAPPDATA%\LarsCloud`. Пароль Google програма не отримує та не зберігає. Токени не записуються до журналу.

## Доступ Google Drive

Програма запитує мінімальну область `drive.file`, а також базові `openid`, `email`, `profile`. `drive.file` дає доступ до файлів і папок, створених або явно відкритих через застосунок, а не довільне читання всього Drive.

## Видалення

За замовчуванням видалення локального файлу не видаляє його резервну копію. Віддалене видалення виконується лише після явного ввімкнення відповідного налаштування. Lar’s Cloud не змінює й не видаляє локальні файли.

Користувач може від’єднати Google-акаунт у налаштуваннях, відкликати доступ у Google Account та видалити локальні службові дані.

## Оновлення

Програма може звертатися до публічного GitHub Releases API для перевірки нової версії. Перед запуском Installer перевіряється SHA-256.

## Контакт

Перед публічним розповсюдженням видавець має замінити цей розділ реальною адресою підтримки та, за потреби, доповнити політику відповідно до законодавства своєї юрисдикції.
